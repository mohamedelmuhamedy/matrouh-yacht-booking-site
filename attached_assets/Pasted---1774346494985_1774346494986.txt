أريد تطوير الموقع الحالي ليصبح منصة سياحة احترافية ومميزة جدًا، مع الحفاظ الكامل على الهوية البصرية الحالية وعدم كسر أي جزء موجود. أريد تنفيذ الميزات التالية داخل المشروع الحالي بشكل منظم وقابل للتوسعة.

المطلوب تنفيذ هذه الـ Features:

==================================================
1) مقارنة بين الباقات (Compare Packages)
==================================================
أضف ميزة مقارنة بين الباقات السياحية.

المطلوب:
- إضافة زر "قارن" على كل باقة
- السماح للمستخدم باختيار حتى 3 باقات للمقارنة
- عند الاختيار، يظهر شريط ثابت أو زر واضح لفتح المقارنة
- إنشاء Compare Modal أو Compare Section تعرض الباقات المختارة جنبًا إلى جنب

يجب أن تعرض المقارنة:
- اسم الباقة
- السعر
- مدة الرحلة
- نوع الرحلة
- الأنشطة المشمولة
- الوجبات
- الانتقالات
- هل مناسبة للعائلات؟
- هل مناسبة للأجانب؟
- مستوى التجربة
- المميزات الرئيسية
- زر "احجز الآن" لكل باقة

ملاحظات:
- على الموبايل اجعل المقارنة scrollable بشكل مرتب
- لا تجعل التصميم مزدحمًا
- اجعل التجربة سهلة وواضحة

==================================================
2) صفحة تفاصيل احترافية لكل رحلة / باقة
==================================================
بدل الاكتفاء بالكارت فقط، أنشئ صفحة تفاصيل مستقلة لكل باقة.

المطلوب داخل صفحة التفاصيل:
- عنوان الرحلة
- Hero image أو gallery
- وصف احترافي
- السعر
- المدة
- شارة مثل: الأكثر طلبًا / مناسبة للعائلات / تجربة فاخرة
- قسم "لماذا تختار هذه الرحلة؟"
- جدول الرحلة (itinerary) يوم بيوم أو خطوة بخطوة
- ماذا تشمل الباقة
- ماذا لا تشمل الباقة
- ماذا تحضر معك
- هل مناسبة للأطفال؟
- سياسة الحجز / الإلغاء
- قسم أسئلة شائعة خاص بكل رحلة
- CTA واضح: احجز الآن / استفسر واتساب
- قسم رحلات مشابهة في الأسفل

تقنيًا:
- إذا كان المشروع React Router أو أي Router، أضف صفحات dynamic route مثل:
  /packages/:slug
- أنشئ بيانات منظمة للباقات بدل النصوص المبعثرة داخل الكومبوننتات
- اجعل كل باقة لها slug خاص بها
- استخدم structure نظيف يسهل إضافة باقات مستقبلًا

==================================================
3) تعدد اللغة + تعدد العملة
==================================================
أريد الموقع يدعم على الأقل:
- العربية
- English

والعملات:
- EGP
- USD
- SAR

المطلوب:
- Language Switcher في الـ navbar
- Currency Switcher أيضًا بشكل متناسق
- حفظ اختيار اللغة والعملة في localStorage
- العربية = RTL
- الإنجليزية = LTR
- عند تغيير اللغة تتغير direction ومحاذاة العناصر بشكل صحيح
- ترجمة النصوص الثابتة بالكامل
- ترجمة labels / placeholders / buttons / badges
- الأسعار تتغير حسب العملة المختارة
- استخدم conversion rates ثابتة داخل config مؤقتًا لسهولة التطوير
- اجعل تنسيق الأسعار احترافي حسب العملة
- لا تكسر التصميم الحالي

البنية المفضلة:
- translations/ar.ts
- translations/en.ts
- currency config file
- language / currency context أو hooks واضحة

==================================================
4) AI Travel Assistant داخل الموقع
==================================================
أريد إضافة مساعد ذكي داخل الموقع يساعد الزوار على اختيار الرحلة المناسبة.

المطلوب:
- Floating chat button أو chat widget متناسق مع التصميم
- UI احترافي يشبه assistant وليس chat بدائي
- يعرض أسئلة سريعة مقترحة مثل:
  - أفضل رحلة للعائلات؟
  - ما الفرق بين الباقات؟
  - ما أفضل رحلة للأجانب؟
  - ما أفضل خيار حسب الميزانية؟
  - ما الأنشطة المناسبة للأطفال؟
- إذا لم يكن هناك backend AI حقيقي الآن، أنشئ Smart Rule-Based Assistant داخل الفرونت أو utility logic
- يعتمد على بيانات الباقات ويقترح الأنسب بناءً على:
  - عدد الأفراد
  - الميزانية
  - نوع الرحلة
  - وجود أطفال
  - هل الزائر محلي أم أجنبي
- عند الوصول لتوصية مناسبة، يعرض:
  - اسم الباقة
  - سبب التوصية
  - السعر
  - زر عرض التفاصيل
  - زر الحجز
  - زر التواصل واتساب

مهم:
- اجعل architecture قابلة لاحقًا للربط مع OpenAI API أو backend AI حقيقي
- افصل logic عن UI
- لا تجعل الشات مزعجًا أو يغطي العناصر المهمة

==================================================
5) Personalized Homepage
==================================================
أريد الصفحة الرئيسية تصبح أكثر ذكاءً وتخصيصًا حسب سلوك المستخدم.

المطلوب:
- إذا كانت أول زيارة: اعرض أشهر الباقات
- إذا عاد المستخدم مرة أخرى: اعرض "أكمل من حيث توقفت"
- إذا شاهد باقة معينة من قبل: اعرضها في قسم "آخر ما شاهدته"
- إذا اختار لغة English: اجعل التوصيات مناسبة للأجانب
- إذا اختار سفاري أو يخت أو عائلي: اعرض هذه الفئة أولًا في الصفحة
- استخدم localStorage لتخزين:
  - آخر الباقات التي تمت مشاهدتها
  - آخر نوع رحلة مفضل
  - آخر لغة
  - آخر عملة
- أضف Sections مثل:
  - Recommended for you
  - Recently viewed
  - Most popular
  - Best for families
  - Best for foreign visitors

مهم:
- لا تجعل الصفحة معقدة
- اجعل التخصيص subtle واحترافي
- إذا لم توجد بيانات سابقة، استخدم fallback جيد

==================================================
6) قسم "Why this trip?" لكل باقة
==================================================
أضف داخل كل باقة وفي صفحة التفاصيل قسم ذكي بعنوان:
- لماذا هذه الرحلة؟
أو بالإنجليزية:
- Why this trip?

المطلوب:
- عرض 3 إلى 5 أسباب قصيرة ومقنعة حسب كل باقة
- أمثلة:
  - مناسبة للعائلات
  - أفضل قيمة مقابل السعر
  - تجربة هادئة ومريحة
  - مناسبة للأجانب
  - تشمل أكثر الأنشطة طلبًا
  - مناسبة للأطفال
  - رحلة قصيرة وسهلة
  - مثالية لمحبي المغامرة

تقنيًا:
- اجعل هذه الأسباب جزءًا من data model لكل باقة
- اعرضها بشكل badge list أو highlighted features
- استخدم نفس البيانات أيضًا داخل AI assistant والمقارنة

==================================================
7) Referral / Loyalty System
==================================================
أريد إضافة نظام ولاء وإحالة بسيط واحترافي داخل الموقع.

المرحلة الأولى المطلوبة:
- قسم واضح بعنوان:
  - برنامج الإحالة
  - أو Earn Rewards / Referral Program
- الفكرة:
  - شارك رابطك
  - عند حجز صديقك تحصل على خصم أو رصيد
- أنشئ UI لعرض:
  - كود إحالة تجريبي
  - عدد الإحالات
  - المكافأة الحالية
  - إجمالي الرصيد
- أضف CTA مثل:
  - انسخ كود الإحالة
  - شارك عبر واتساب
- أضف قسم Loyalty بسيط:
  - عميل جديد
  - عميل مميز
  - VIP traveler
- كل مستوى له مميزات شكلية أو خصومات توضيحية

تقنيًا:
- لا حاجة لربط backend حقيقي الآن
- استخدم mock data قابلة للربط لاحقًا
- أنشئ الهيكل البرمجي بحيث يمكن لاحقًا ربطه بقاعدة بيانات ومستخدمين حقيقيين

==================================================
8) متطلبات هيكلية مهمة جدًا
==================================================
- لا تكسر أي جزء من الموقع الحالي
- لا تغيّر الهوية البصرية الحالية
- حسّن الكود ونظمه قدر الإمكان
- افصل البيانات عن الواجهة
- أنشئ types/interfaces واضحة إذا كان المشروع TypeScript
- اجعل كل Feature قابلة للتوسعة مستقبلًا
- راعِ الـ responsive behavior على desktop / tablet / mobile
- لا تضع كل شيء في ملف واحد
- أضف reusable components عند الحاجة

==================================================
10) المطلوب النهائي بعد التنفيذ
==================================================
بعد إنهاء العمل:
- أعطني ملخصًا واضحًا بكل الملفات التي تم تعديلها أو إنشاؤها
- اشرح كيف أضيف باقة جديدة
- اشرح كيف أضيف لغة جديدة
- اشرح كيف أضيف عملة جديدة
- اشرح كيف أربط AI Assistant لاحقًا بـ API حقيقي
- اشرح كيف أربط نظام الإحالة لاحقًا بقاعدة بيانات

ابدأ التنفيذ على مراحل منظمة داخل المشروع الحالي، مع الحفاظ على التصميم الحالي وبدون تكسير أي شيء موجود.
