أريد تنفيذ التعديلات التالية فقط داخل المشروع الحالي، مع الحفاظ على التصميم الحالي وعدم كسر أي جزء موجود:

==================================================
1) نقل برنامج الإحالة والمكافآت خارج الصفحة الرئيسية
==================================================
المشكلة:
حاليًا برنامج الإحالة والولاء ظاهر كسكشن داخل منتصف الصفحة الرئيسية، وهذا غير مناسب لأنه يقطع تسلسل الصفحة الأساسية.

المطلوب:
- إزالة قسم برنامج الإحالة والولاء من الصفحة الرئيسية بالكامل
- لا أريد أي teaser
- لا أريد banner
- لا أريد section صغير
- لا أريد preview داخل الـ homepage نهائيًا

أريده فقط كميزة مستقلة يمكن الوصول لها من الأعلى.

التنفيذ المطلوب:
- أضف زر أو رابط واضح في الـ navbar باسم:
  - "المكافآت"
  - أو "الإحالة والمكافآت"
- عند الضغط عليه يفتح صفحة مستقلة مثل:
  - /rewards
  - أو /referral
- انقل صفحة/محتوى برنامج الإحالة الحالي بالكامل إلى هذه الصفحة المستقلة
- احتفظ بنفس تصميم صفحة الإحالة الحالية:
  - كود الإحالة
  - الرصيد
  - عدد الإحالات
  - المستويات
  - زر المشاركة عبر واتساب
- لكن لا تضعه داخل الصفحة الرئيسية مرة أخرى بأي شكل

==================================================
2) إصلاح مشكلة زر المقارنة
==================================================
المشكلة:
زر "قارن" لا يعمل بشكل صحيح حاليًا.

المطلوب:
- زر "قارن" على كل باقة يجب أن:
  - يضيف الباقة إلى قائمة المقارنة
  - ولا يفتح صفحة الحجز
  - ولا ينقل المستخدم إلى أي route خاطئ
- السماح بإضافة حتى 3 باقات فقط للمقارنة
- عند اختيار باقة أو أكثر يظهر:
  - compare bar
  - أو زر واضح لفتح المقارنة
- عند الضغط على المقارنة:
  - تفتح Compare Modal
  - أو Compare Drawer / Compare Section مستقل
- يجب أن تعمل المقارنة فعليًا وتعرض البيانات الصحيحة لكل باقة

مهم جدًا:
- افصل event handling بشكل صحيح
- لو الكارت كله clickable أو داخل Link، لا تجعل زر المقارنة يرث نفس السلوك
- استخدم stopPropagation / preventDefault إذا لزم الأمر
- تأكد أن زر المقارنة له click behavior مستقل تمامًا
- لا تجعل الضغط على "قارن" يفتح الحجز أو التفاصيل

==================================================
3) إصلاح مشكلة زر التفاصيل
==================================================
المشكلة:
عند الضغط على "التفاصيل" أو أي زر متعلق بالتفاصيل، يتم فتح صفحة الحجز بدل صفحة التفاصيل.

المطلوب:
- زر "التفاصيل" يجب أن يفتح صفحة تفاصيل الباقة فقط
- لا يفتح صفحة الحجز
- لا يمرر المستخدم إلى booking section
- لا ينفذ أي action خاص بالحجز

التنفيذ المطلوب:
- إذا كانت هناك صفحة details جاهزة، اربط زر "التفاصيل" بها
- وإذا لم تكن موجودة بشكل صحيح، أنشئ/أصلح route مثل:
  /packages/:slug
- كل باقة يجب أن تفتح صفحة تفاصيلها الصحيحة حسب الـ slug
- صفحة التفاصيل تعرض بيانات الرحلة الخاصة بهذه الباقة فقط

مهم جدًا:
- لو الكارت أو الزرار ملفوف داخل Link أو onClick عام يرسل للحجز، افصل هذا السلوك
- اجعل:
  - "احجز الآن" = يذهب للحجز فقط
  - "التفاصيل" = يذهب لصفحة التفاصيل فقط
  - "قارن" = يضيف للمقارنة فقط
- لا أريد أي تداخل بين الأزرار الثلاثة

==================================================
4) مراجعة سلوك الكروت والأزرار
==================================================
أريد مراجعة كاملة لسلوك عناصر الباقات الحالية لأن الواضح أن هناك تداخل في الـ click actions.

راجِع الآتي:
- هل الكارت كله clickable؟
- هل يوجد Link wrapper حول عناصر داخلية؟
- هل يوجد onClick على parent container يلتقط كل الضغطات؟
- هل الأزرار الداخلية ترث navigation غير مقصود؟
- هل يوجد overlay أو absolute layer يسرق الضغطات؟
- هل يوجد pointer-events أو z-index عاملين مشكلة؟

المطلوب النهائي:
- الضغط على "احجز الآن" فقط يفتح الحجز
- الضغط على "التفاصيل" فقط يفتح التفاصيل
- الضغط على "قارن" فقط يضيف للمقارنة
- لا يوجد أي click conflict
- لا يوجد أي event bubbling يسبب تنقل خاطئ

==================================================
5) الحفاظ على التصميم الحالي
==================================================
- لا تغيّر الهوية البصرية
- لا تغيّر ترتيب الصفحة الرئيسية إلا بإزالة قسم الإحالة منها
- لا تعدّل النصوص بدون داعٍ
- لا تكسر الـ responsive design
- لا تغيّر شكل الباقات إلا بالقدر اللازم لإصلاح السلوك

==================================================
6) المطلوب بعد التنفيذ
==================================================
بعد الانتهاء:
- أعطني ملخصًا سريعًا بالملفات التي تم تعديلها
- واذكر بالضبط:
  - كيف تم إصلاح زر المقارنة
  - كيف تم إصلاح زر التفاصيل
  - أين أصبحت صفحة المكافآت الآن
  - ما هو الـ route الخاص بها
