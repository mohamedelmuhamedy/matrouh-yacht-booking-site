ممتاز. الأساس اتبنى، لكن قبل اعتبار النظام جاهز أريد تنفيذ مرحلة Hardening + Production Readiness + Admin Polish.

نفّذ التعديلات التالية فقط على النظام الحالي بدون كسر ما تم بناؤه:

==================================================
1) تأمين نظام الأدمن بشكل صحيح
==================================================
المشكلة الحالية:
لا أريد أي secrets أو بيانات دخول تكون hardcoded داخل الكود.

المطلوب:
- انقل JWT secret إلى environment variable
- لا تضع أي secret افتراضي داخل الكود
- اجعل admin credentials لا تكون hardcoded
- أنشئ admin user داخل قاعدة البيانات
- خزّن كلمة المرور بشكل hashed وآمن
- أضف session / token validation محترمة
- أضف logout واضح
- أضف حماية لكل admin routes
- إذا كان token منتهي أو غير صالح:
  - يتم تسجيل الخروج تلقائيًا
  - والرجوع لصفحة login

مهم:
- لا أريد user/password ثابتين داخل الواجهة أو الكود
- أريد طريقة واضحة لإضافة admin جديد لاحقًا

==================================================
2) تحسين إدارة الرحلات والبيانات
==================================================
أريد تحسين CRUD الخاص بالباقات ليكون Production-ready.

المطلوب:
- validation قوي لكل الحقول
- منع حفظ رحلة بدون:
  - عنوان
  - slug
  - سعر
  - وصف أساسي
- التحقق أن slug فريد
- منع القيم غير المنطقية للأسعار
- منع reviewCount أو rating غير الصحيح
- دعم draft / published / archived status
- دعم duplicate package
- دعم soft delete أو archive بدل الحذف النهائي إذا أمكن
- إضافة last updated timestamp
- إضافة created at / updated at في الجداول

==================================================
3) تحسين إدارة الصور والفيديوهات
==================================================
أريد نظام media أوضح وأقوى.

المطلوب:
- إذا كانت الصور حاليًا مجرد روابط نصية، أضف دعم رفع ملفات حقيقي
- دعم:
  - multiple image upload
  - delete image
  - reorder images
  - cover image selection
- دعم video URL على الأقل بشكل منظم
- preview واضح للميديا داخل الأدمن
- تحقق من نوع الملف وحجمه
- لا تترك ملفات مكررة أو بيانات media غير مرتبطة

إذا كان المشروع يحتاج storage solution:
- اختر الحل الأنسب للمشروع الحالي
- ووصّله بشكل منظم

==================================================
4) إعدادات الموقع والمرونة
==================================================
أريد تحسين settings حتى تكون عملية فعلًا.

المطلوب:
- site title
- meta description
- default currency
- exchange rates
- WhatsApp number
- phone number
- social links
- business hours
- logo / branding assets
- hero content
- feature toggles:
  - AI assistant on/off
  - compare on/off
  - testimonials loop on/off
  - rewards page on/off

==================================================
5) تحسين إدارة الحجوزات
==================================================
أريد قسم الحجوزات يكون أنضف وأكثر فائدة.

المطلوب:
- البحث بالاسم أو الهاتف
- الفلترة بالحالة
- الترتيب بالأحدث
- عرض العملة والسعر وقت الطلب إذا أمكن
- زر واتساب مباشر
- update status:
  - جديد
  - تم التواصل
  - مؤكد
  - ملغي
- export CSV إذا كان سهل التنفيذ
- notes داخلية للأدمن إن أمكن

==================================================
6) تحسين تجربة الأدمن
==================================================
أريد Admin UI أوضح وأسهل.

المطلوب:
- dashboard فيها:
  - عدد الرحلات
  - عدد الحجوزات
  - الحجوزات الجديدة
  - الرحلات المنشورة
  - آخر التعديلات
- forms مرتبة
- loading states
- success/error toasts
- confirm dialog قبل الحذف
- unsaved changes warning قبل مغادرة الصفحة
- mobile usability معقولة للـ admin أيضًا

==================================================
7) ربط الموقع العام بالبيانات بشكل كامل
==================================================
أريد التأكد أن الموقع العام لا يعتمد على بيانات hardcoded في الأجزاء الأساسية.

تحقق من أن:
- الرحلات في الصفحة الرئيسية تأتي من DB
- صفحات التفاصيل تأتي من DB
- الأسعار تأتي من DB
- testimonials تأتي من DB
- settings الأساسية تأتي من DB
- أي تعديل من /admin ينعكس على الموقع العام

==================================================
8) تحسينات مهمة قبل الإطلاق
==================================================
أضف ما أمكن من الآتي:
- seed script واضح
- migration files مرتبة
- error handling محترم في الـ API
- empty states
- 404 للباقة غير الموجودة
- fallback للصورة إذا لم توجد
- sanitize / validate للمدخلات
- منع كسر الصفحة إذا كانت البيانات ناقصة

==================================================
9) المطلوب النهائي بعد التنفيذ
==================================================
بعد الانتهاء:
- أعطني ملخصًا واضحًا بما تم تحسينه
- اذكر الملفات التي تم تعديلها
- اشرح لي:
  - كيف أضيف admin جديد
  - كيف أغيّر كلمة المرور
  - كيف أرفع صور وفيديوهات
  - كيف أنشر / أخفي رحلة
  - كيف أعدل سعر رحلة
  - كيف أغيّر رقم الواتساب
  - كيف أشاهد الحجوزات
- واذكر بوضوح أي جزء ما زال تجريبيًا ويحتاج تطوير لاحق
