هناك مشكلة مهمة في لوحة الأدمن الحالية: الصفحات لا تُظهر البيانات الحالية الموجودة فعليًا داخل النظام، وهذا يجعل التعديل غير عملي.

أريد إصلاح هذا السلوك فقط، بدون تغيير الهوية البصرية.

==================================================
1) صفحة تعديل الباقة يجب أن تكون pre-filled
==================================================
المشكلة الحالية:
عند فتح صفحة تعديل باقة مثل:
/admin/packages/:id/edit
تظهر الفورم فارغة تقريبًا، بينما المفروض تكون كل الحقول مليانة ببيانات الباقة الحالية.

المطلوب:
- عند فتح صفحة تعديل أي باقة:
  - اعمل fetch لبيانات الباقة الحالية من قاعدة البيانات / API
  - املأ كل الحقول تلقائيًا بالقيم الحالية
- يشمل ذلك:
  - slug
  - titleAr / titleEn
  - shortDescriptionAr / shortDescriptionEn
  - fullDescriptionAr / fullDescriptionEn
  - category
  - duration
  - location
  - الأسعار
  - badges
  - whyThisTrip
  - includes / excludes
  - itinerary
  - faq
  - الصور
  - الفيديوهات
  - status
  - featured / popular / active
  - أي حقول أخرى موجودة في الباقة

مهم:
- لا أريد نموذجًا فارغًا عند التعديل
- إذا كانت البيانات موجودة في DB يجب أن تظهر كلها
- لو بعض الحقول فقط غير موجودة فعلًا، هذه فقط تبقى فارغة
- أضف loading state واضحة أثناء تحميل بيانات الباقة
- أضف error state لو فشل الجلب
- تأكد أن الحفظ بعد التعديل لا يمسح الحقول القديمة بسبب initial values خاطئة

==================================================
2) صفحة التقييمات يجب أن تعرض كل التقييمات، وليس 5 فقط
==================================================
المشكلة الحالية:
صفحة /admin/testimonials تعرض 5 تقييمات فقط، بينما أريد رؤية كل التقييمات الموجودة في قاعدة البيانات.

المطلوب:
- اجلب كل testimonials الموجودة فعليًا من قاعدة البيانات
- لا تعتمد على limit ثابت = 5 إلا إذا كان هناك pagination حقيقية
- إذا كان العدد كبيرًا:
  - أضف pagination
  - أو load more
  - أو table / grid مع search
لكن يجب أن أستطيع الوصول لكل التقييمات، وليس أول 5 فقط

أريد أيضًا:
- total count صحيح
- search بالاسم أو نص التقييم إذا أمكن
- filter بالحالة:
  - ظاهر
  - مخفي
- ترتيب بالأحدث أو sortOrder إذا موجود
- عند الضغط على "تعديل" يفتح بيانات التقييم الحالية pre-filled أيضًا

مهم:
- لو هناك hardcoded mock testimonials، أزل الاعتماد عليها
- صفحة الأدمن يجب أن تقرأ من البيانات الحقيقية فقط

==================================================
3) صفحة الإعدادات يجب أن تكون مليانة بالقيم الحالية
==================================================
المشكلة الحالية:
صفحة /admin/settings تظهر الحقول فارغة، بينما المفروض تظهر فيها الإعدادات الحالية المستخدمة فعليًا في الموقع.

المطلوب:
- عند فتح صفحة الإعدادات:
  - اجلب site settings الحالية من قاعدة البيانات
  - املأ جميع الحقول بالقيم الحالية
- يشمل مثلًا:
  - اسم الشركة بالعربي والإنجليزي
  - الموقع
  - أرقام التواصل
  - واتساب
  - روابط السوشيال
  - العملة الافتراضية
  - أسعار الصرف
  - hero title / subtitle
  - feature toggles
  - أي إعدادات أخرى محفوظة

مهم:
- لا أريد form فارغة
- أريد رؤية القيم الحالية ثم تعديلها
- لو بعض settings keys غير موجودة، أنشئ fallback منظم لكن لا تجعل كل الصفحة blank
- إذا كانت الإعدادات مخزنة على شكل key-value، اعمل mapping صحيح من keys إلى form fields
- عند الحفظ:
  - update الإعدادات الحالية
  - لا تنشئ duplicate records بدون داعٍ
  - لا تمسح قيم موجودة بسبب mapping ناقص

==================================================
4) توحيد سلوك صفحات التعديل
==================================================
أريد نفس المنطق في كل صفحات الأدمن:
- Edit page = current values visible
- Settings page = current saved values visible
- Testimonials edit = current testimonial values visible
- Package edit = current package values visible

يعني أي صفحة "تعديل" أو "إدارة" لازم تبدأ بقراءة البيانات الحالية وملء النموذج بها.

==================================================
5) راجع سبب المشكلة في الكود
==================================================
أريدك أن تراجع وتصلح السبب الحقيقي، والذي غالبًا واحد أو أكثر من التالي:
- fetch لا يعمل عند mount
- id أو slug لا يتم تمريره بشكل صحيح
- response يصل لكن لا يتم map إلى state
- defaultValues في الفورم لا تتحدث بعد وصول البيانات
- useEffect dependencies ناقصة
- controlled inputs غير مربوطة بالبيانات
- settings mapping بين key-value والواجهة غير صحيح
- testimonials API عليها limit ثابت
- البيانات في الصفحة ما زالت تعتمد على mock data

أصلح السبب من الجذر، وليس workaround شكلي.

==================================================
6) تحسين UX في الأدمن
==================================================
أضف الآتي:
- loading spinner أو skeleton أثناء تحميل البيانات
- رسالة واضحة إذا لم توجد بيانات
- رسالة خطأ إذا فشل الجلب
- success toast بعد الحفظ
- لا تجعل الصفحة تبدو فارغة بدون تفسير

==================================================
7) اختبارات مطلوبة بعد الإصلاح
==================================================
اختبر الآتي فعليًا:
- افتح تعديل باقة موجودة وتأكد أن كل الحقول الحالية ظهرت
- عدّل قيمة واحفظ وتأكد أن التعديل انعكس
- افتح صفحة التقييمات وتأكد أن كل التقييمات ظهرت أو يمكن الوصول إليها عبر pagination
- افتح تعديل تقييم وتأكد أن البيانات الحالية ظاهرة
- افتح صفحة الإعدادات وتأكد أن كل القيم الحالية ظاهرة
- عدّل إعدادًا واحفظ وتأكد أنه انعكس على الموقع العام

==================================================
8) المطلوب بعد التنفيذ
==================================================
بعد الانتهاء:
- أعطني ملخصًا واضحًا:
  - لماذا كانت صفحة تعديل الباقة فارغة
  - لماذا كانت صفحة الإعدادات فارغة
  - لماذا كانت صفحة التقييمات تعرض 5 فقط
  - كيف تم إصلاح كل نقطة
- واذكر الملفات التي تم تعديلها
