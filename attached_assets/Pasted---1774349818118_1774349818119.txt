أريد إصلاح مشاكل الوظائف الحالية في الموقع بدون تغيير الهوية البصرية، مع تنفيذ التعديل الخاص بصفحة المكافآت.

المطلوب تنفيذ الآتي فقط:

==================================================
1) نقل "المكافآت" خارج الصفحة الرئيسية
==================================================
- احذف قسم "المكافآت / الإحالة والولاء" من الصفحة الرئيسية بالكامل
- لا أريد أي teaser
- لا أريد أي banner
- لا أريد أي preview داخل الـ homepage
- أريده فقط كرابط أو زر مستقل في الـ navbar باسم:
  "المكافآت"
- عند الضغط عليه يفتح صفحة مستقلة مثل:
  /rewards
- انقل صفحة المكافآت الحالية بالكامل إلى هذا الـ route
- احتفظ بنفس تصميم محتواها الحالي، لكن خارج الصفحة الرئيسية

==================================================
2) إصلاح كروت الباقات لأنها حاليًا لا تعمل
==================================================
المشكلة الحالية:
- الكروت أصبحت مجرد شكل
- الضغط على الأزرار لا يفعل شيئًا
- لا يفتح الحجز
- لا يفتح التفاصيل
- لا يفتح المقارنة

أريد مراجعة كاملة لسلوك كروت الباقات وكل الأزرار بداخلها.

افحص وأصلح الآتي:
- هل يوجد overlay أو layer فوق الكروت يمنع الضغط؟
- هل يوجد pointer-events: none على الكروت أو الأزرار أو parent container؟
- هل يوجد z-index خاطئ أو absolute element يغطي الأزرار؟
- هل يوجد parent onClick / Link / wrapper يعطل الأزرار؟
- هل يوجد button type أو anchor behavior خاطئ؟
- هل الـ handlers غير مربوطة؟
- هل state الخاصة بالمقارنة أو الحجز أو navigation لا يتم استدعاؤها؟
- هل تم كسر الأحداث بعد تعديلات اللغة / العملة / personalization / rewards؟

المطلوب النهائي:
يجب أن تصبح كل كروت الباقات interactive مرة أخرى بشكل كامل.

==================================================
3) السلوك المطلوب لكل زر داخل كل باقة
==================================================

أ) زر "اختر" أو "احجز الآن":
- يجب أن يفتح الحجز فقط
- إما:
  - يفتح booking section
  - أو booking modal / drawer
  - أو يعمل scroll إلى نموذج الحجز
- لكن يجب أن يمرر بيانات الباقة المختارة بشكل صحيح
- لازم أعرف من أي باقة تم الضغط على "اختر"
- يجب أن يظهر اسم الباقة المختارة داخل نموذج الحجز أو booking state

ب) زر "التفاصيل":
- يجب أن يفتح صفحة تفاصيل الباقة فقط
- لا يفتح الحجز
- لا يعمل scroll للحجز
- لا يتجاهل الضغط
- يجب أن يذهب إلى route مثل:
  /packages/:slug
- وكل باقة تفتح صفحة تفاصيلها الصحيحة حسب الـ slug

ج) زر "قارن":
- يجب أن يضيف أو يزيل الباقة من المقارنة
- لا يفتح الحجز
- لا يفتح التفاصيل
- لا يتجاهل الضغط
- يسمح باختيار حتى 3 باقات فقط
- عند اختيار باقة أو أكثر يجب أن يظهر compare bar أو compare action واضح
- وعند فتح المقارنة تظهر بيانات الباقات المختارة بشكل صحيح

==================================================
4) لا أريد الكارت كله يفتح أي شيء تلقائيًا
==================================================
المطلوب:
- لا تجعل الكارت كله clickable إذا كان هذا سيسبب تعارض
- يمكن أن يكون الكارت static
- لكن الأزرار الثلاثة داخله يجب أن تعمل 100%:
  - اختر / احجز
  - التفاصيل
  - قارن

إذا كان جعل الكارت clickable يسبب conflict:
- ألغِ clickability عن الكارت نفسه
- واجعل فقط الأزرار الداخلية هي المسؤولة عن كل action

==================================================
5) إصلاح مشكلة event conflicts
==================================================
أريد معالجة أي تعارض في الضغطات بين:
- parent card
- inner buttons
- links
- wrappers
- motion / animation layers
- compare state
- booking state
- routing

استخدم ما يلزم مثل:
- stopPropagation
- preventDefault
- فصل الـ handlers
- إزالة أي Link wrapper غير مناسب
- إزالة أي layer تغطي الأزرار
- ضبط z-index
- ضبط pointer-events
- التأكد أن كل زر له handler مستقل وصحيح

==================================================
6) اختبار كل الباقات فعليًا
==================================================
بعد الإصلاح، اختبر كل الباقات واحدة واحدة وتأكد من الآتي:

لكل باقة:
- زر "اختر" يفتح الحجز
- زر "التفاصيل" يفتح صفحة التفاصيل
- زر "قارن" يضيف للمقارنة

اختبر على:
- desktop
- mobile

وتأكد أن:
- لا يوجد زر ميت
- لا يوجد ضغط لا يفعل شيئًا
- لا يوجد route خاطئ
- لا يوجد تعارض بين الأزرار

==================================================
7) الحفاظ على التصميم الحالي
==================================================
- لا تغيّر الألوان
- لا تغيّر الهوية البصرية
- لا تغيّر شكل الكروت إلا إذا كان ذلك ضروريًا جدًا لإصلاح الـ interaction
- لا تعيد قسم المكافآت إلى الصفحة الرئيسية
- لا تلمس أجزاء أخرى بدون داعٍ

==================================================
8) المطلوب بعد التنفيذ
==================================================
بعد الانتهاء:
- أعطني ملخصًا واضحًا بالملفات التي تم تعديلها
- واشرح باختصار:
  - أين كانت مشكلة الضغطات
  - كيف تم إصلاح زر الحجز
  - كيف تم إصلاح زر التفاصيل
  - كيف تم إصلاح زر المقارنة
  - ما هو route صفحة المكافآت الجديدة
