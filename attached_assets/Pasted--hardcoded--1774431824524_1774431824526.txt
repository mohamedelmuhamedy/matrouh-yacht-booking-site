أريد تحويل الموقع الحالي من موقع يعتمد على بيانات hardcoded داخل الكود إلى موقع مُدار من خلال لوحة تحكم احترافية داخل نفس المشروع.

الهدف:
إنشاء Admin Panel كاملة على route:
- /admin

بحيث الشخص المسؤول عن الموقع يقدر يعدّل كل المحتوى القابل للتعديل بدون الرجوع للكود أو إعادة النشر في كل مرة.

==================================================
1) المطلوب الأساسي
==================================================
أريد إنشاء لوحة تحكم حقيقية داخل المشروع الحالي، وليس مجرد صفحة شكلية.

الموقع العام يجب أن يقرأ بياناته من قاعدة بيانات أو مصدر بيانات منظم، وليس من objects hardcoded داخل الكومبوننتات.

لوحة التحكم /admin يجب أن تسمح بالتعديل على:
- الرحلات / الباقات
- الأسعار
- الصور
- الفيديوهات
- النصوص الأساسية
- آراء العملاء
- إعدادات الموقع
- العملات
- بيانات التواصل
- محتوى الصفحة الرئيسية
- محتوى صفحات التفاصيل

==================================================
2) الحماية والدخول
==================================================
أريد صفحة /admin تكون محمية.

المطلوب:
- صفحة login للأدمن
- لا يمكن فتح /admin إلا بعد تسجيل الدخول
- حماية الـ routes الخاصة بالأدمن
- session أو auth مناسب
- لو لا توجد بنية auth حاليًا، أضف أبسط وأوضح نظام آمن متوافق مع المشروع الحالي
- لا تجعل لوحة التحكم public

إذا كان الأنسب استخدام:
- Supabase Auth
- Firebase Auth
- أو auth بسيط على backend
اختر الحل الأنسب للمشروع الحالي، لكن نفّذه بشكل منظم وقابل للتوسعة.

==================================================
3) إدارة الرحلات / الباقات
==================================================
أنشئ داخل لوحة التحكم قسمًا لإدارة الرحلات بشكل كامل.

المطلوب لكل رحلة:
- إضافة رحلة جديدة
- تعديل رحلة موجودة
- حذف رحلة
- إخفاء / إظهار الرحلة
- ترتيب ظهور الرحلات
- تحديد هل الرحلة:
  - featured
  - popular
  - family-friendly
  - foreigner-friendly
  - active / inactive

الحقول المطلوبة لكل رحلة:
- id
- slug
- titleAr
- titleEn
- shortDescriptionAr
- shortDescriptionEn
- fullDescriptionAr
- fullDescriptionEn
- category
- duration
- location
- rating
- reviewCount
- badges
- whyThisTrip
- suitableFor
- includes
- excludes
- itinerary
- faq
- bookingPolicy
- cancellationPolicy
- featured
- popular
- active
- sortOrder

==================================================
4) إدارة الأسعار والعملات
==================================================
أريد الأدمن يقدر يعدّل الأسعار بسهولة.

المطلوب:
- سعر أساسي للرحلة بالمصري EGP
- دعم العملات:
  - EGP
  - USD
  - SAR
- اجعل النظام يدعم أحد الخيارين:
  1) auto conversion من السعر الأساسي
  2) manual override لكل عملة إذا أراد الأدمن إدخال سعر خاص
- داخل لوحة التحكم، الأدمن يقدر:
  - يغيّر السعر بالمصري
  - يغيّر سعر الدولار
  - يغيّر سعر الريال
  - أو يضبط conversion rate عامة من إعدادات الموقع

مهم:
- الموقع العام يعرض السعر حسب العملة المختارة
- التعديل في الأدمن ينعكس على الموقع مباشرة
- راعِ تنسيق الأسعار جيدًا

==================================================
5) إدارة الصور والفيديوهات
==================================================
أريد الأدمن يقدر يرفع ويعدل ميديا كل رحلة بسهولة.

المطلوب:
- رفع صور متعددة لكل رحلة
- حذف الصور
- إعادة ترتيب الصور
- تحديد صورة رئيسية cover image
- رفع فيديوهات أو إضافة video URL
- حذف الفيديوهات
- دعم gallery داخل صفحة تفاصيل الرحلة

مهم:
- استخدم تخزين منظم للملفات
- لا تضع الصور hardcoded داخل الكود
- اربط الصور والفيديوهات بالرحلة المناسبة
- راعِ preview داخل لوحة التحكم

- CTA

11) تجربة استخدام لوحة التحكم
==================================================
أريد لوحة التحكم تكون سهلة وواضحة، وليست معقدة.

المطلوب:
- dashboard رئيسية فيها إحصائيات سريعة
- sidebar واضح
- forms مرتبة
- validation جيد
- success / error messages
- loading states
- image previews
- confirm dialog قبل الحذف
- حفظ التعديلات بشكل واضح
- لا تجعل الأدمن يحتاج فهم تقني

==================================================
12) متطلبات مهمة جدًا في التنفيذ
==================================================
- لا تكسر الموقع الحالي
- لا تغيّر الهوية البصرية العامة للموقع العام
- افصل admin عن public UI
- أنشئ data layer نظيفة
- حوّل البيانات الحالية الموجودة في الموقع إلى قاعدة بيانات أو seed data منظمة
- اجعل الموقع العام يقرأ من قاعدة البيانات بدل البيانات الثابتة
- لا تترك أي جزء مهم hardcoded إذا كان من المفترض تعديله من الأدمن
- اجعل النظام قابلًا لإضافة رحلات كثيرة مستقبلًا

==================================================

==================================================
أنشئ models / tables / collections مناسبة على الأقل لـ:
- admins

- testimonials
- bookings
- site_settings
- homepage_sections
- faqs
- currencies أو exchange_rates

إذا كان هناك schema أو migrations مطلوبة، أنشئها بشكل منظم.

==================================================
14) صلاحيات وملاحظات مهمة
==================================================
أريد على الأقل:
- admin authenticated access
- حماية من أي تعديل غير مصرح
- عدم السماح لأي زائر عادي بالوصول إلى لوحة التحكم
- رفع الصور والفيديوهات بشكل آمن
- التحقق من نوع وحجم الملفات
- منع كسر البيانات بسبب إدخال ناقص أو خاطئ

==================================================
15) تحسينات إضافية أريدها إذا كانت مناسبة
==================================================
إذا أمكن، أضف أيضًا:
- draft / publish status للرحلات
- duplicate package
- preview before save أو preview route
- last updated timestamp
- sort order drag-and-drop للصور أو الرحلات
- archive بدل delete النهائي لبعض العناصر
- restore أو soft delete لو أمكن

==================================================
16) مهم جدًا بخصوص التطوير
==================================================
لا أريد مجرد UI فقط.
أريد النظام فعليًا شغال end-to-end:
- الأدمن يعدّل
- البيانات تتحفظ
- الموقع العام يتحدث بناءً على البيانات الجديدة

اختر أفضل بنية مناسبة للمشروع الحالي دون تكسير المشروع.
إذا احتجت backend أو database integration، نفّذها بشكل منظم داخل نفس المشروع.

==================================================
17) المطلوب النهائي بعد التنفيذ
==================================================
بعد الانتهاء:
- أعطني ملخصًا واضحًا بكل الملفات التي تم تعديلها أو إنشاؤها
- اشرح لي كيف أدخل إلى /admin
- اشرح كيف أضيف رحلة جديدة
- اشرح كيف أعدل السعر
- اشرح كيف أرفع صور وفيديوهات
- اشرح كيف أعدل النصوص الأساسية
- اشرح كيف أرى الحجوزات
- اشرح كيف أضيف مستخدم admin لاحقًا
